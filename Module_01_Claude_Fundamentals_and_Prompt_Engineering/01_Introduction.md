# 🔰 Introduction — Claude Fundamentals and Prompt Engineering

## 1. What Learners Should Understand

Claude is an AI assistant that can help with:

```text
✅ Writing
✅ Summarization
✅ Brainstorming
✅ Research support
✅ Document analysis
✅ Coding support
✅ Business planning
✅ Learning support
```

Claude is powerful, but learners should understand:

```text
❌ Claude is not always correct.
❌ Claude may sound confident even when information is incomplete.
❌ Claude should not replace human review in high-stakes work.
✅ Important outputs need verification.
```

---

## 2. Claude as an Assistant, Not a Search Engine

Learners should understand this difference:

| Tool | Main Purpose |
|---|---|
| Search engine | Finds web pages and links |
| Claude | Understands, drafts, explains, reasons, summarizes, and creates structured outputs |

---

## 3. Claude Use Cases

| Use Case | Example |
|---|---|
| Writing | Emails, reports, proposals |
| Learning | Topic explanations, quizzes, lesson plans |
| Analysis | Compare options, summarize risks |
| Productivity | Meeting notes, schedules, task plans |
| Coding | Explain code, debug, create examples |

---

## 4. Prompt Engineering

Prompt engineering means giving Claude clear instructions.

A weak prompt:

```text
Write about marketing.
```

A stronger prompt:

```text
Act as a B2B marketing strategist.

Create a 30-day LinkedIn content plan for a SaaS company selling HR software.

Return a table with:
1. Day
2. Post topic
3. Hook
4. Audience
5. CTA
```

---

## 5. The CRAFT Prompting Method

Learners should use this structure:

| Letter | Meaning |
|---|---|
| C | Context |
| R | Role |
| A | Action |
| F | Format |
| T | Tone / Tests |

Example:

```text
Context:
I run a small training company.

Role:
Act as a senior course designer.

Action:
Create a 1-day Claude training workshop plan.

Format:
Return a table with session, topic, activity, and duration.

Tone:
Professional and practical.
```

---

## 6. Common Beginner Mistakes

```text
❌ Asking vague questions
❌ Not giving context
❌ Not specifying output format
❌ Asking too many tasks in one prompt
❌ Not checking accuracy
❌ Not asking Claude to state assumptions
```

---

## 7. Best Practice Rule

Learners should remember:

```text
Better input ➜ Better output
Clear context ➜ Better reasoning
Defined format ➜ Better usability
Verification ➜ Safer results
```
