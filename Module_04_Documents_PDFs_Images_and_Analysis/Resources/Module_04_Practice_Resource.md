# 🔶 Module 4 Practice Resource — Vendor Proposal Review

## Scenario

A fictional company, NorthStar Retail, is reviewing a proposal from a delivery vendor called QuickRoute Logistics.

## Proposal Summary

QuickRoute Logistics offers delivery support for NorthStar Retail’s local e-commerce orders.

## Proposed Services

```text
- Same-city package pickup
- Local delivery within 24–48 hours
- Return pickup support
- Weekly delivery summary report
- Basic customer support for delivery status
```

## Proposed Timeline

| Stage | Timeline |
|---|---|
| Contract discussion | Week 1 |
| System setup | Week 2 |
| Pilot launch | Week 3 |
| Full rollout | Week 6 |

## Cost Summary

```text
Monthly platform fee: $1,500
Per-delivery charge: $2.20
Return pickup charge: $1.80
Setup fee: Not specified
Penalty terms: Not specified
```

## Service Level Notes

```text
- Vendor states that most deliveries are completed within 24–48 hours.
- No exact refund process is provided.
- No detailed escalation process is provided.
- No data privacy terms are included in this summary.
```

## Defect / Complaint Trend Table

| Month | Late Deliveries | Missing Package Complaints | Damaged Package Complaints |
|---|---:|---:|---:|
| January | 18 | 4 | 6 |
| February | 25 | 5 | 7 |
| March | 32 | 8 | 9 |
| April | 28 | 6 | 10 |

## Internal Concerns

```text
- The proposal does not mention penalty terms.
- Data privacy details are missing.
- Return pickup process is not fully explained.
- Escalation ownership is unclear.
- Setup fee is not specified.
```

## Practice Goal

Learners should ask Claude to:

```text
☑️ Summarize the proposal
☑️ Extract key terms
☑️ Identify missing information
☑️ Analyze complaint trend data
☑️ Create an action plan
```
