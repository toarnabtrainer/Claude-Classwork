# 🔰 Introduction — Evaluation, Safety, Privacy, and Responsible Use

## 1. What Learners Should Understand

AI outputs should be reviewed, especially when used for real work.

Claude can help with drafting, analysis, and planning, but learners should understand:

```text
🔴 AI can be wrong.
🔴 AI can omit important information.
🔴 AI can produce unsupported claims.
✅ Humans should verify important outputs.
```

---

## 2. Evaluation Means Checking Quality

A good Claude output should be checked for:

```text
☑️ Accuracy
☑️ Completeness
☑️ Format compliance
☑️ Tone
☑️ Practical usefulness
☑️ Safety
☑️ Source grounding
```

---

## 3. Success Criteria

Before judging an output, learners should define what success means.

Example:

```text
A good customer support response should:
- Summarize the issue correctly
- Use polite tone
- Avoid unsupported promises
- Ask for missing information
- Escalate risky cases
```

---

## 4. Hallucination Risk

A hallucination is when an AI output contains information that is unsupported or incorrect.

Learners should reduce this risk by using prompts such as:

```text
Do not invent facts.
Use only the provided document.
If information is missing, say "Not found."
Separate facts from assumptions.
```

---

## 5. Privacy and Sensitive Data

Learners should avoid sharing:

```text
❌ Passwords
❌ API keys
❌ Private customer data
❌ Medical details
❌ Bank details
❌ Confidential company strategy
❌ Employee records
```

Use fictional or anonymized data for demos.

---

## 6. Safety Across Claude Features

| Feature | Main Safety Concern |
|---|---|
| Projects | Uploading sensitive or outdated knowledge |
| Artifacts | Publishing unsupported or public content |
| Design | Creating misleading visuals or brand-inaccurate assets |
| Skills | Uploading unsafe scripts or instructions |
| Connectors | Granting access to external systems and private data |
| API/tools | Automated actions without approval |

---

## 7. Responsible Use Rule

Learners should remember:

```text
Claude assists.
Humans verify.
Critical decisions need expert review.
```
