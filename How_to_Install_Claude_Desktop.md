# 💻 How to Install Claude Desktop

This document explains how learners can install Claude Desktop and prepare a Windows system for development-related Claude workflows.

---

## 🔰 What Learners Should Understand

Claude Desktop is the desktop version of Claude. It may be useful for workflows involving desktop features, extensions, local MCP servers, Cowork, or advanced development demonstrations.

Official installation reference:

```text
https://support.claude.com/en/articles/10065433-install-claude-desktop
```

---

## ✅ Windows Installation Steps

1. Open the Claude download page:

```text
https://claude.com/download
```

2. Select:

```text
Windows
```

3. Download the installer.

4. Open the downloaded installer file.

5. Complete installation.

6. Open Claude from:

```text
Start Menu ➜ Claude
```

7. Sign in with your Claude account.

---

## ✅ macOS Installation Steps

1. Open:

```text
https://claude.com/download
```

2. Select:

```text
macOS
```

3. Download the installer.

4. Open the downloaded file.

5. Move Claude to Applications if prompted.

6. Open Claude from:

```text
Applications ➜ Claude
```

7. Sign in with your Claude account.

---

## ✳️ Windows Developer Mode Step

For development-related workflows, local app testing, Claude Code, extensions, or advanced demos, Windows users may need Developer Mode enabled.

Use this path:

```text
Settings ➜ System ➜ Advanced ➜ For developers ➜ Developer Mode ➜ ON
```

Learners should understand:

```text
✅ Developer Mode may require administrator access.
✅ Organization-managed devices may block this option.
✅ This step is normally not required for simple Claude chat usage.
```

Official Microsoft reference:

```text
https://learn.microsoft.com/en-us/windows/advanced-settings/developer-mode
```

---

## ✨ First Test After Installation

Use this prompt in Claude Desktop:

```text
Explain what Claude Desktop can help me do in a professional training workflow.
Return the answer in 5 bullet points.
```

---

## 🔶 Feature Visibility Check

After signing in, learners should check whether these are visible:

```text
☑️ New Chat
☑️ Projects
☑️ Code
☑️ Customize
☑️ Design, if available
☑️ Artifacts, if available
☑️ Connectors under Customize
☑️ Skills under Customize
```

Availability can vary by plan, organization settings, region, and current product rollout.

---

## ✴️ Important Note About Artifacts

Artifacts may be available in two ways:

```text
1. Dedicated Artifacts space in the Claude sidebar
   ✅ Best for classroom demonstration
   ✅ Use: Artifacts ➜ New Artifact

2. Chat-based Artifact creation
   ✳️ Claude may create an Artifact automatically when it generates substantial standalone content.
   ✳️ In that case, the Artifact may appear in a dedicated window beside the conversation.
```

For predictable class practice, learners should use:

```text
Artifacts ➜ New Artifact
```

---

## 🔴 Troubleshooting

| Problem | Suggested Fix |
|---|---|
| Claude Desktop does not open | Restart the system and reinstall |
| Sign-in fails | Check internet, browser, and account credentials |
| Design not visible | Check plan and organization settings |
| Skills not visible | Check Customize section and plan availability |
| Connectors not visible | Check account and organization connector settings |
| Developer Mode unavailable | Device may be managed by organization policy |
