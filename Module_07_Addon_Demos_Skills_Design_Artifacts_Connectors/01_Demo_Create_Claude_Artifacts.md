# ✨ Demo 1 — How to Create Claude Artifacts

## 1. What Learners Should Understand

Artifacts are substantial standalone outputs that Claude can help create, revise, and reuse.

Examples:

```text
✅ FAQ guide
✅ Dashboard
✅ HTML page
✅ Checklist
✅ Mini tool
✅ Report
✅ Template
```

---

## 2. Correct Classroom Method

For reliable practice, use:

```text
Claude.ai ➜ Artifacts ➜ New Artifact
```

Learners should understand that this is more predictable than asking from a normal chat.

---

## 3. Alternative Method

Claude may also create an Artifact automatically from a normal chat when the output is substantial.

However:

```text
✳️ Chat-based Artifact creation may not always trigger the Artifact workspace.
✅ For class practice, use Artifacts ➜ New Artifact.
```

---

## 4. Resource to Use

Open:

```text
Resources/EcoServe_Addon_Demo_Resources.md
```

Use the section:

```text
Artifact Source Brief
```

Copy or upload that content into the New Artifact workflow.

---

## 5. Artifact Creation Steps

```text
1. Open Claude.ai
2. Click Artifacts in the left sidebar
3. Click New Artifact
4. Paste or upload the Artifact Source Brief
5. Paste the Artifact Creation Prompt below
6. Review the generated Artifact
7. Revise the Artifact using the revision prompt
```

---

## 6. Artifact Creation Prompt

```text
Create a reusable customer-facing FAQ Artifact for EcoServe.

Audience:
EcoServe customers and support agents.

Purpose:
Help users understand EcoServe plans, pickup requests, reporting, escalation, and unsupported areas.

Use only the uploaded or pasted source content.

Include:
1. Clear title
2. Getting started section
3. Plan comparison table
4. Pickup request steps
5. Pickup status explanation
6. Reporting overview
7. Escalation guidance
8. Unsupported areas
9. Missing information section
10. Quality checklist

Rules:
- Do not invent pricing.
- Do not provide legal compliance advice.
- Do not promise vendor actions unless confirmed.
- Do not claim EcoServe guarantees certification or regulatory compliance.
- Mark missing information clearly.
```

---

## 7. Artifact Revision Prompt

Continue in the same Artifact workflow:

```text
Review this Artifact for:

1. Accuracy against source content
2. Customer friendliness
3. Reusability
4. Missing information
5. Unsupported claims
6. Clarity of escalation guidance

Then revise it without adding unsupported facts.

At the end, include:
- What changed
- What information is still missing
- Final quality checklist
```

---

## 8. Strong Output Characteristics

A good Artifact should include:

```text
✅ Clear title
✅ Structured sections
✅ Customer-friendly wording
✅ Missing information section
✅ No invented pricing
✅ No unsupported legal/compliance guarantees
✅ Reusable format
```

---

## 9. Deleting Artifacts

Learners should understand:

```text
To remove an Artifact created from a chat, delete the entire associated chat from chat history.
```

Important caution:

```text
🔴 If the Artifact was published or shared, use the visible Unpublish or Unshare option where available before assuming the shared version is no longer accessible.
```

---

## 10. Troubleshooting

| Problem | Suggested Fix |
|---|---|
| Artifact does not appear from chat | Use Artifacts ➜ New Artifact |
| Output contains unsupported claims | Add stricter source rules |
| Artifact is too generic | Add audience, purpose, and required sections |
| Artifact is hard to reuse | Ask Claude to convert it into a reusable template |
| Published Artifact needs removal | Unpublish/unshare first where available |
