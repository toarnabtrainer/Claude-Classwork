# 🔰 Introduction — Productivity and Business Workflows

## 1. What Learners Should Understand

Claude can help convert messy information into useful workplace outputs.

Examples:

```text
✅ Raw meeting notes ➜ action items
✅ Customer complaint ➜ professional response
✅ Business idea ➜ execution plan
✅ Long notes ➜ executive summary
✅ Task list ➜ prioritized schedule
```

---

## 2. Common Productivity Use Cases

| Use Case | Example Output |
|---|---|
| Email drafting | Polite follow-up, apology email, proposal email |
| Meeting notes | Summary, decisions, action items |
| Reports | Executive summary, risk report, weekly update |
| Planning | Daily plan, weekly plan, project timeline |
| Decision support | Pros/cons table, scoring matrix |
| Brainstorming | Campaign ideas, process improvements |

---

## 3. Workplace Prompting Rule

Learners should give Claude:

```text
☑️ Business context
☑️ Audience
☑️ Desired output
☑️ Tone
☑️ Constraints
☑️ Review criteria
```

---

## 4. Human Review Reminder

For workplace use, learners should remember:

```text
🔴 Claude can draft.
✅ Humans approve.
🔴 Claude can suggest.
✅ Humans decide.
🔴 Claude can summarize.
✅ Humans verify important facts.
```

---

## 5. Reusable Workflow Thinking

A workflow prompt is not a one-time question. It is a repeatable process.

Example:

```text
Input: customer complaint
Claude output:
1. Summary
2. Sentiment
3. Category
4. Suggested reply
5. Escalation required
```

Learners should build repeatable prompts for tasks they do often.
